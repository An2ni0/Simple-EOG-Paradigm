# EOG 眼电网格采集系统用户手册 & 操作 SOP

本系统是专为 **眼电 (EOG) 与眼球注视点同步采集** 设计的简化版双机系统。系统通过局域网 UDP 传输进行同步打标，用于获取眼电信号并用于后续注视轨迹建模及注意力时间分析。

---

## 一、 系统架构与网络配置

系统由两台电脑组成，必须在同一局域网下，并使用**有线网卡**连接交换机以防无线丢包与延迟。

```
【 范式控制 PC 】                                   【 cDAQ 采集 PC 】
   运行: 眼动范式.py                                   运行: DAQ_GUI_server.py
   功能: 渲染5x5网格红点                              功能: 连续记录 EOG 电压
        记录本地 CSV 日志                                  根据 UDP 指令自动存盘并打标
        发送 UDP 网络指令  ──────── 局域网有线 ────────►   生成 .bin 与 _meta.json 
```

### 1. 网络连通度配置（首次部署执行）
*   **采集 PC 端 (DAQ)**：建议将有线网卡设为静态 IP（例如 `10.10.10.100`）。
*   **范式 PC 端**：
    *   在 [common.py](file:///c:/Users/simia/OneDrive/Data/DoCs/Tools/Simple_EOG_Paradigm/common.py) 的第 14 行，确保 IP 与采集端一致：
        `DAQ_PC_IP = os.environ.get("DAQ_PC_IP", "10.10.10.100")`
*   **防火墙设置**：采集端 PC 必须在防火墙中开放 **UDP 端口 `55555`** 的入站规则。

## 二、 医生与学生的实验操作 SOP（核心步骤）

请严格按照以下步骤开展实验，切勿颠倒顺序：

### 步骤 1：启动 cDAQ 采集软件（在 cDAQ PC 上操作）
1.  双击运行或在命令行运行：
    ```bash
    python DAQ_GUI_server.py
    ```
2.  确认窗口正常开启，顶部状态栏显示 **“🟢 预览模式 (等待 CMD_START 指令)”**，且界面四周呈绿色。此时波形实时滚动，说明硬件连接与电极贴放正常。

### 步骤 2：启动范式主程序（在范式控制 PC 上操作）
1.  在范式电脑运行：
    ```bash
    python 眼动范式.py
    ```
2.  **录入信息**：屏幕中央会首先弹出对话框，提示 **“请输入患者姓名/编号”**。
3.  输入患者信息（如 `001_张三`），点击“确定”。

### 步骤 3：确认设备联动与实验准备
1.  弹窗关闭后，范式电脑将切入全屏黑底网格界面，显示 **“按回车键开始实验”**。
2.  **检查联动状态**：此时，范式计算机会自动发送启动信号。**请扭头检查 cDAQ PC 的屏幕**：
    *   cDAQ GUI 的状态栏必须变成 **“🔴 正在录制中 (RECORDING)”**，且窗口四周呈现**红色边框**。
    *   如果未变红，说明网络未通或防火墙拦截，请勿开始实验！

### 步骤 4：正式执行实验
1.  指导志愿者做好准备，双眼注视屏幕。
2.  在范式电脑上按下 **“回车键”**。
3.  **志愿者任务**：
    *   当屏幕中出现**红色圆点**时，请用眼睛紧盯着红点；
    *   当红点消失、屏幕中心出现**白色十字**时，请闭眼休息或注视中心十字。
4.  实验会依次遍历 5x5 网格的 25 个位置（自动跳过最中央的中心格），每个位置默认采集 2 次。

### 步骤 5：实验结束与存盘
1.  25 个点位全部遍历完成后，范式电脑会播报语音“实验完成”，并安全退出全屏。
2.  cDAQ PC 收到自动发出的停止命令，界面恢复为 **“🟢 绿色预览状态”**，并提示数据已成功落盘。

## 三、 实验中的键盘快捷控制

在范式 PC 上，操作人员可通过键盘随时掌控实验进度：
*   `空格键`：**暂停**实验（屏幕提示“已暂停”）。再次按 `回车键` 即可在当前位置继续实验。
*   `ESC 键`：**强行中止**实验。程序会立刻向 cDAQ 发送停止信号并安全退出，**已采集的数据段仍会安全存盘**，不会丢失。
*   `回车键`：在暂停状态下用于**恢复实验**，以及在手动确认模式下切换方向。

## 四、 实验数据文件位置

1.  **范式控制端 (Paradigm PC)**:
    *   本地行为日志保存在 `logs/` 文件夹下。
    *   命名格式：`[姓名]_[任务名]_[年月日_时分秒].csv`。
    *   内含每一步骤的绝对时间戳与相对实验开始的相对毫秒数（例如：`TARGET_START` 代表红点出现）。
2.  **cDAQ 采集端 (DAQ PC)**:
    *   数据保存在软件同目录下的 `EOG/` 文件夹下。
    *   `.bin` 文件：包含 hEOG、vEOG 等所有模拟通道的原始二进制电压。
    *   `_meta.json` 文件：包含当时接收到的 UDP 触发包记录（包含事件名称和精确的 cDAQ 物理采样点索引 `daq_sample_index`）。

## 五、 参数微调与志愿者反应迟缓调整指南

对于老年人、病患或反应较慢的志愿者，系统支持在 [眼动范式.py](file:///c:/Users/simia/OneDrive/Data/DoCs/Tools/Simple_EOG_Paradigm/%E7%9C%BC%E5%8A%A8%E8%8C%83%E5%BC%8F.py) 的顶部（第 28~34 行）直接修改核心参数：

```python
# ========================== 实验核心参数 ==========================
GRID = 5                  # 支持 3 / 5 等奇数网格
TARGET_SHOW = 1.0         # 目标点显示时长（秒）
REST_TIME = 2.0           # 每次试次间休息时长（秒）
REPEAT_PER_CELL = 2       # 每个点位采集次数
DIRECTION_REST = 3.0      # 自动切换方向休息时长(秒)
MANUAL_CONFIRM_DIRECTION = False  # 切换新方向是否需要医生按回车手动确认
```

### 调整方案建议：
1.  **红点注视时间不够**：
    将 `TARGET_SHOW` 延长至 `1.5` 或 `2.0` 秒，让反应迟缓的患者在目光移过去后，有足够长的时间（> 1秒）保持稳定注视。
2.  **前一次运动残余影响下一次**：
    将 `REST_TIME` 延长至 `2.5` 或 `3.0` 秒，使患者的眼睛能够从边缘完全回弹并稳定注视在中心十字，确保建模基线彻底归零。
3.  **患者极度疲劳或无法配合连续自动测试**：
    将 `MANUAL_CONFIRM_DIRECTION` 修改为 `True`。
    *   **效果**：系统在切换网格的每一个新方向前，都会在屏幕提示“准备下一个方向”，并播放语音，**此时实验无限期挂起等待**。
    *   医生可以口头指导患者做好准备，等患者准备妥当后，**医生按下回车键**，范式才会开始该方向的红点呈现。这能极大地给患者提供喘息和调整时间。

## 六、 后续建模与数据分析对齐指南 (给分析人员)

本系统使用 UDP 对齐方案：

1.  **读取元数据**：
    打开 `EOG/` 文件夹下的 `_meta.json`，在 `events` 列表中可以读到范式 PC 广播的打标点。例如：
    ```json
    {
        "event": "T_1_R0C1_TARGET_START",
        "system_time": 17839304.51,
        "daq_sample_index": 25000
    }
    ```
    代表在 cDAQ 采集到第 `25000` 个数据点时，网格 `(Row 0, Col 1)` 位置的红点亮起。

2.  **基线漂移扣除 (Baseline Correction)**：
    由于眼电容易随时间产生基线漂移，在提取看红点段的电压时：
    *   提取 `REST_START` 至 `REST_END` 期间（注视中心十字）的 hEOG 和 vEOG 均值电压，作为基线 $V_{base\_h}, V_{base\_v}$；
    *   提取 `TARGET_START` 至 `TARGET_END` 期间（注视目标红点）的电压 $V_h, V_v$；
    *   使用相对差值进行建模：$\Delta V_h = V_h - V_{base\_h}$，$\Delta V_v = V_v - V_{base\_v}$。

3.  **数据切片抗反应延迟限制**：
    *   由于受试者在红点亮起后，通常需要 `150ms ~ 300ms` 的反应时间（扫视期）眼球才真正动过去。
    *   **建议分析窗**：提取 `TARGET_START` **后延 350ms** 至 `TARGET_END` 的数据切片作为稳定期进行平均，以剔除反应时（Reaction Time）对注视空间建模的干扰。
